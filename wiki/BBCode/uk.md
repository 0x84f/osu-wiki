# BBCode

**BBCode** це [Мова розмітки](https://uk.wikipedia.org/wiki/Мова_розмітки_даних) яка використовується на форумах osu! і, по великому рахунку, на більшості форумах в інтернеті. Вона використовується для широких можливостей форматування тексту. Ця мова складається з тегів, які записуються по обидві сторони тексту, вказуючи на його форматування, атрибутів, вкладень та багато іншого. Вона використовується в багатьох місцях на сайті osu! наприклад в публікаціях на форумах, підписах, сторінках користувачів та описах мап.

![Редактор публікацій на форумах з його кнопками](img/editor-uk.jpg?1 "Редактор публікацій на форумах")

## Використання

Натиснувши кнопку розмітки без виділення тексту, створиться набір відкритих і закритих тегів на місці текстового курсора у вікні редактора. Виділивши текст перед натисканням кнопки розмітки, створить теги з обох сторін цього тексту.

Користувачі, які хочуть застосувати декілька тегів до одного і того ж тексту, можуть зробити це вклавши теги один в одного, Проте, послідовність і спосіб вкладання цих тегів **є важливим** при їх об'єднанні. При неправильному використанні, форматування буде невдале.  

Нижче показано правильне і неправильне використання вкладених тегів:

- `[centre][b]text[/b][/centre]` Правильно
- `[b][centre]text[/b][/centre]` Неправильно

## Теги

BBCode, як і багато інших мов розмітки, форматує текст використовуючи систему тегів, які позначаються парою квадратних дужок (`[]`). Ці теги розділені на "відкриваючі" та "закриваючі", які відрізняються додаванням слешу (`/`). Якщо точніше, то закриваючі теги мають слеш відразу після відкриваючої дужки, в той час як відкриваючі теги не містять слешу.

Також, важливо відмітити, що відкриваючі теги іноді мають в собі знак рівності (`=`), щоб вказати на посилання, розміри шрифту, та інші подібні елементи.

Нижче, перераховані та описані в деталях, BBCode теги, які підтримуються сайтом osu!

### Жирний шрифт

```
[b]text[/b]
```

Тег `[b]` використовується, щоб надати виразності тексту за допомогою жирного шрифту. Цей тег не змінює розмір шрифту.

Кнопка на панелі форматування: ![Жирний шрифт](img/bold.png "Жирний")

### Курсив

```
[i]text[/i]
```

Тег `[i]` використовується, щоб злегка надати виразності тексту, нахиливши його.

Кнопка на панелі форматування ![Курсив](img/italic.png "Курсив")

### Підкреслення

```
[u]text[/u]
```

Тег `[u]` використовується, щоб надати виразності тексту, намалювавши горизонтальну лінію під ним. Ця лінія буде модифікована іншими тегами, такими як жирний шрифт і курсив.

### Перекреслення

```
[strike]text[/strike]
```

Тег `[strike]` використовується, щоб відмітити видалення попередньо написаного тексту, за допомогою горизонтальної лінії, яка "перекреслює" текст.

Кнопка на панелі форматування: ![Перекреслення](img/strike.png "Перекреслення")

### Колір

```
[color=HEXCODE]text[/color]
```

*Щоб дзінатися всі кольори, перегляньте сторінку [назви кольорів в X11](https://uk.wikipedia.org/wiki/%D0%9A%D0%BE%D0%BB%D1%8C%D0%BE%D1%80%D0%B8_HTML#%D0%9D%D0%B0%D0%B7%D0%B2%D0%B8_%D0%BA%D0%BE%D0%BB%D1%8C%D0%BE%D1%80%D1%96%D0%B2_%D0%B2_X11)*

Тег `[color]` використовується, щоб стилізувати текст різними кольорами. Тег використовує формат [HEX коду](https://en.wikipedia.org/wiki/Web_colors#Hex_triplet), щоб вказати колір, також можна використовувати назви кольорів HTML, такі як "red" або "green". Щоб вказати колір, замініть аргумент `HEXCODE` на відповідний HEX код або HTML назву.

Вищевказаний аргумент не повинен мати лапки (`"`), і не має стандартного кольору. Якщо ніякий аргумент не вказано, або використовуються лапки, тег не буде вважатися тегом BBCode.

### Розмір шрифту

```
[size=NUMBER]text[/size]
```

Тег `[size]` використовується, щоб стилізувати текст шляхом застосування різних розмірів шрифту. В даний час, osu! веб сайт підтримує чотири розміри шрифту: 50, 85, 100 і 150. На сайті розміри називаються: "дрібний", "малий", "звичайний" та "великий" відповідно.

Аргумент `NUMBER` не приймає лапки, а приймає лише один із чотирьох розмірів, які підтримуються. Якщо введене число не є одним з чотирьох, які підтримуються, текст повернеться до стандартного роміру.

Кнопка на панелі форматування: ![Розмір шрифту](img/font-size-uk.png "Розмір шрифту")

### Спойлер

*Не плутати з [Spoilerbox](#спойлербокс).*

```
[spoiler]text[/spoiler]
```

Тег `[spoiler]` використовується, щоб закрити небажану для деяких людей інформацію, суцільним чорним кольором, який розкриває прихований текст при його виділені. Якщо об'єднати з тегом [`[color]`](#колір), чорне покриття не буде змінено, але текст під ним буде кольоровим, незважаючи чи можливо його прочитати чи ні.

Тег найчастіше вживається для запобігання виказання критичної/чутливої інформації про ТБ шоу, фільм або інші медіа. Також, це іноді використовується для комедійного ефекту або виділення. 

### Контейнер

*Не плутати з [Spoilerbox](#спойлербокс).*

```
[box=NAME]
text
[/box]
```

Тег `[box]` використовується, щоб сховати текст і картинки в посилання, яке можна натиснути. При натисканні, вміст буде показаний по аналогії з випадаючим меню.

Текст посилання вказується аргументом `NAME`. Вказавши цей аргумент ви створите заголовок всередині контейнера. Розмір контейнера буде підлаштований відповідно до заголовку. Якщо `NAME` не вказано, тоді тег `[box]` створить контейнер без заголовку. Аргумент не використовує лапки (`"`), і буде замінювати їх на пробіли.

Тег переважно використовується, щоб сховати величезну кількість тексту і картинок, які можуть занадто збільшити розмір публікації. Часто можна побачити в FAQ або [skin](/wiki/Skinning) реліз постах.

*Зауважте: На панелі форматування, кнопка яка створює контейнер, хоч і називається "Спойлер", вона не створює `[spoilerbox]`*

Кнопка на панелі форматування: ![Контейнер](img/spoilerbox.png "Контейнер")

### Спойлербокс

```
[spoilerbox]text[/spoilerbox]
```

*Spoilerbox* це особливий тип контейнера, який немає аргументу `NAME`. Заголовок цього контейнера, завжди показується як `SPOILER`. Спойлербокси мають свій власний тег (`[spoilerbox]`), проте їх функціонал є ідентичним з тегом [контейнер](#контейнер).

### Цитата

```
[quote="NAME"]
text
[/quote]
```

Тег `[quote]` використовується для оформлення довгих цитат за допомогою відступів, кольору, жирності та відокремлення тексту за подопомогою рожевої вертикальної лінії. Вміст цитати розміщений між відкриваючими і закриваючими тегами. В той час як аргумент `NAME` вказує на автора цитати (проте цей аргумент не обов'язковий). Текст в середині цитати враховує всі пробіли та переноси на новий рядок.

*Зауважте: аргумент `NAME` має бути в лапках (`"`).*

Довгі цитати зазвичай використовуються замість однолінійних цитат в більш формальних розмовах, коли цитата має довжину три або більше рядки. Проте, на форумах osu! вони зазвичай використовуються для відповіді на коментар іншого користувача, що може бути зроблено автоматично за допомогою кнопки Але всередині форумів вони використовуються щоб відповісти іншому гравцю, що можна робити автоматично за допомогою кнопки `Відповісти з цитуванням`, яка знаходиться в верхньому правому куті коментаря (показано нижче). Проте ця кнопка **з'явиться лише тоді, коли курсор наведений на коментар**.

![Кнопка цитати](img/quotereply.png)

### Код

```
[code]
text
[/code]
```

Тег `[code]` використовується щоб створити *преформатовані блоки коду* (a.k.a. *preformatted text*).На сайті, `[code]` завжди буде форматований моноширинним шрифтом всередині напівпрозорого сірого контейнеру. Форматування тексту всередині тега буде вважатися окреми текстом і не буде частиною коду.

В форумах, код часто використовується щоб опублікувати код для [storyboard](/wiki/Storyboard), або в туторіалах яким треба показати якийсь код.

### Центрування

```
[centre]text[/centre]
```

Тег `[centre]` використовується для вирівнювання тексту всередині контейнера. Використовується для стилістичного ефекту в заголовках.
Якщо тег розташований всередині `[quote]`, цитата буде відцентрована, але стилістичні лінії не будуть змінені.

### Посилання

```
[url=LINK]text[/url]
```

Тег `[url]` використовується щоб вставити посилання.

*Зауважте: цей тег використовується щоб зробити посилання із своєю назвою, наприклад: [текст замість адреси](#)*

Щоб створити посилання за допомогою `[url]`, користувач повинен зазначити два аргументи назва, і посилання. Текст має бути вказаний між тегами, а посилання в аргументі `LINK`, без лапків (`"`). Якщо назва не вказана, буде відображено адресу посилання.

Кнопка на панелі форматування: ![Посилання](img/url.png "Посилання")

### Профіль

```
[profile=userid]username[/profile]
```

Тег `[profile]` використовується щоб додати посилання на профіль користувача по його нікнейму або ID. `[profile]` і `[url]` відрізняються тим що `[profile]` відображає картку профіля коли посилання виділене.

*Зауважте: ID користувача - це цифри після `/users/` в посиланні на профіль.*

Якщо було вказано тільки ID користувача, то треба вписати текст між тегами, він не буде відображений, замість нього буде відображено ім'я користувача. Якщо ви вкажете ім'я користувача, то після зміни імені посилання перестане працювати.

### Форматовані списки

```
[list=TYPE]
[*]item 1
[*]item 2
[*]item 3
[/list]
```

Тег `[list]` використовується для вставки одного із двох типів списків по всьому сайту, використовуючи (`[*]`) ви створюєте новий елемент в списку (показано вище). По замовчуванню це створить простий дотований текст.

Якщо `TYPE` вказаний (не важливо, що саме ви напишете), буде створено нумерований список.

*Зауважте: форматовані списки можуть бути накладеними та вкладеними, але це може викликати проблеми з форматуванням.*

Кнопка на панелі форматування: ![Список](img/list.png "Список") ![Пронумерований список](img/list-numbered.png "Пронумерований список")

### Email

```
[email=ADDRESS]text[/email]
```

Tег `[email]` створює посилання, яке направляє на стандартну поштову систему з шаблоном листа на вказану пошту.

Щоб створити посилання, треба вказати два аргументи: `ADDRESS` з дійсним адресом і `text` з текстом посилання. Якщо `text` не вказаний посилання не буде відображатися правильно.

### Картинки

```
[img]ADDRESS[/img]
```

Тег `[img]` використовуються для вкладення картинок з інтернету в пости. Щоб використовувати тег користувач повинен вставити поссилання на картинку замість `ADDRESS`. Локальні файли
(e.g. `C:\Users\Name\Pictures\image.jpg`) **не працюють**.

*Зауважте: Посилання на сайт це **не** теж саме що і посилання на зображення.*

Щоб дізнатися посилання на картинку, потрібно натиснути правою кнопкою мишки по картинці і вибрати `Копіювати адресу зображення`.

Хоч картинки й можна брати де завгодно, ми рекомендуємо публікувати картинку на довірених сайтах, таких як [Imgur](https://imgur.com), тому що деякі сайти не дозволяють використовувати прямі посилання на картинки.

Кнопка на панелі форматування: ![Зображення](img/image.png "Зображення")

### YouTube

```
[youtube]VIDEO_ID[/youtube]
```

тег `[youtube]` використовується для вкладення [YouTube](https://youtube.com) відео на сайті. Тегу потрібно тільки ID відео (**не** все посилання) замість `VIDEO_ID`.

ID відео знаходиться після `v=` в посиланні на відео.

### Звуки

```
[audio]URL[/audio]
```

Тег `[audio]` використовується для вкладення [HTML5](https://en.wikipedia.org/wiki/HTML5) аудіо програвача. Файли можуть бути взяти звідки завгодно, якщо файл знаходиться в заданому користувачем посиланні. Локальні файли
(e.g. `C:\Users\Name\Music\audio.mp3`) **не працюють**.

*Увага: Не всім сайтам з музикою подобається використання їх музики на сайті через занепокоєння про піратство. osu! не відповідає за неправомірне використання музичних треків.*

Щоб вкласти файл таким способом, користувач повинен вставити посиллання (e.g. `https://www.example.com/example.mp3`) між `[audio]` тегами.

### Заголовок (v1)

```
[heading]text[/heading]
```

Тег `[heading]` використовується для форматування тексту в великий, рожевий заголовок. Тег не підтримує багаторівневі заголовки.

Кнопка на панелі форматування: ![Заголовок](img/heading.png "Заголовок")

### Помітка

```
[notice]
text
[/notice]
```

Тег `[notice]` використовується щоб помістити параграф в великий, виділений, і темний контейнер. В основному використовується для поміток та застережень про деякі теми.

## Застарілі теги

Наступні теги колись використовувалися по сайту, але не використовуються зараз. Вони описані нижче для мети збереження історії.

### Google

```
[google]search query[/google]
```

Тег `[google]` це застарілий тег для вставки пошукових запитів.

Тег перекидав користувачів на сайт Google з вказаним пошуковим запитом, але оскільки користувач заходить зі свого аккаунту, результати будуть різними.

### Lucky

```
[lucky]search query[/lucky]
```

Тег `[lucky]` є таким же самим як і тег `[google]`

### Заголовок (v2)

```
[text]
```

Тег *Heading (v2)* є застарілим тегом, який який форматував текст в химерний, рожевий, підкреслений заголовок. Тег працював тільки в форумах карт і з'являвся тільки після публікації. В тега не було власної кнопки і він був записаний за допомогою лапок
